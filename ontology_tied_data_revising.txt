Your task is to validate and correct the dataset of 100 ontology‑grounded QA examples.

I am giving you as context:
A TTL ontology file (already fully materialized; treat it as complete truth).

The JSON array of 100 questions.

Process each question independently.

VALIDATION RULES
For each question:

1. Reasoning ↔ Ontology consistency
Every class, superclass, property, or relation mentioned in the reasoning must appear in the TTL ontology.
If the reasoning references something not present in the TTL, it is inconsistent.

2. Gold mapping ↔ Reasoning consistency
The gold_mapped_answer must include all and only the ontology elements explicitly referenced in the reasoning.
If the reasoning mentions a superclass, it must appear in the mapping.
If the mapping contains elements not mentioned in the reasoning, it is inconsistent.

3. Answer ↔ Reasoning consistency
The generated answer must logically follow from the reasoning.
If not, it is inconsistent.

4. No external inference
Do not infer anything beyond what is explicitly present in the TTL file.

CORRECTION RULES
If a question is inconsistent:

Fix only the fields that are wrong, keeping the rest unchanged.

You may correct:

the reasoning

the generated answer

the gold_mapped_answer

Corrections must strictly follow the TTL ontology.

Do not invent ontology facts.
