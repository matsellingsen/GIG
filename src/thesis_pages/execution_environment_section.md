#### Execution environment
The system runs entirely in a Python‑based runtime and is executed locally to maximize reproducibility, privacy, and independence from external APIs. All inference is performed using the OpenVINO runtime on a Lenovo Yoga Slim 7 equipped with an Intel® Core™ Ultra 7 258V processor, integrated Intel NPU, and 32 GB of RAM. This hardware configuration is important because OpenVINO’s NPU backend is only supported on specific Intel architectures, and the available NPU memory and execution model directly constrain the feasible model size and quantization format. The use of a small, locally hosted model is therefore both a design choice and a hardware‑driven requirement, enabling low‑latency inference without relying on a GPU or cloud resources. Dependencies are isolated within a dedicated virtual environment with pinned package versions to ensure stable and repeatable execution across machines and over time. The system follows a strictly sequential and stateless execution model: each agent runs independently and communicates exclusively through structured JSON input/output, ensuring clean handoffs, predictable behavior, and full traceability of the inference process.
